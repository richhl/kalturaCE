Overview

The following documentation provides all of the details a web developer will need when looking to use Kaltura's Application Programming Interfaces (APIs).
Kaltura's APIs allow partners to interact with the Kaltura open source video platform through web-services. API's are based on the REST principles

Each request can be an HTTP POST or GET, and the response is a gziped content (assuming that the client specified that gzip is supported)

Each Kaltura API method is located at a unique URI:
http://www.kaltura.com/api_v3/?service=[SERVICENAME]&action=[ACTIONNAME]
Where [SERVICENAME] is the name of the API service and [ACTIONNAME] is the action you would like to perform

Note that before using the Kaltura APIs, you must apply for a Partner ID. have a Partner ID. If you are using the Kaltura Community Edition, this can be a local Partner ID on your server, or you may choose to create a Kaltura Partner ID, which will allow you to interact with Kaltura's system going forward as well.

For additional support please visit our Community Forums.

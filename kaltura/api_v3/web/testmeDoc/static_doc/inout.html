General API Input/Output

  • The input parameters should be sent as a url encoded key-value string, e.g:
    id=abc&name=name%20with%20spaces

  • When the input parameter is an object, e.g. entry, it must be flattened as followed
    entry:id
    entry:name
    entry:source

    The object will be marked as objectName:*, e.g. entry:*

  • Date format is YYYY-MM-DD HH:MM:SS (2008-09-29 10:46:36)

  • General API input parameters:
    Field Name Data Type Description
    ks String Kaltura session, if required by the method
  • API Output Container

    Each API response is wrapped in a special container object with the following properties: "result" & "error".

    • result - the action return value (can be simple or complexed object)
    • error - an array of errors (If no errors occurred, it will be empty), each error has 2 properties "code" and "desc"

    Sample Successful result:

    <xml> <result> <executionTime>0.313575983047</executionTime> <theResultOject> <someProperty></someProperty> <anotherProperty></anotherProperty> </theResultObject> </result> </xml>

    Sample failed result:

    <xml> <result> <error> <code>ERROR_CODE</code> <message>Error description</desc> </error> </result> <executionTime>0.270357131958</executionTime> </xml>
  • General API errors
    Error code Error description
    MISSING_KS Missing KS. Session not established
    INTERNAL_SERVERL_ERROR Internal server error
    SERVERL_ERROR Server error %ERROR_DESCRIPTION%
    INVALID_KS Invalid KS [%KS%]. Error [%KS_ERROR_CODE%,%KS_ERROR_DESCRIPTION%]
    MANDATORY_PARAMETER_MISSING Mandatory paramter missing [%PARAMETER_NAME%]
